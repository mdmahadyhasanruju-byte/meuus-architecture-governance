## 2. Governance Boundary

- These rules do not approve publication.
- These rules do not approve implementation.
- These rules do not approve deployment.
- These rules do not create final copy.
- These rules do not make pages canonical.
- These rules do not verify vocabulary.
- These rules do not verify Arabic/Quran/Seerah content.
- These rules do not verify legal, financial, psychological, medical, recovery, religious, or scientific claims.
- These rules only define safe writing standards for future public-preview drafts.

Core rule:
Public page writing must be truthful, humble, educational, clearly labeled, and bounded by evidence. Every page must separate what exists from what is draft, prototype, future, or requiring review.

## 3. Voice and Tone Rules

The writing voice must be:
- plain
- calm
- honest
- humble
- human
- educational
- beginner-friendly
- evidence-aware
- spiritually respectful without becoming religious authority
- professional enough for public trust
- simple enough for students and citizens

The writing voice must not be:
- hype-based
- sales-heavy
- manipulative
- overconfident
- guru-like
- clinical
- legal-authority style
- religious-authority style
- financial-advice style
- diagnostic
- emotionally coercive
- inflated startup language

## 4. Universal Page Writing Structure

Every public-preview page should follow:

- Page title
- Status label
- Level label
- Review status
- Source/provenance status
- Short purpose
- What this page is
- What this page is not
- Simple explanation
- Why this matters
- Common misunderstanding
- Basic framework
- Real-life example
- Reflection question
- One small action
- Related pages
- Required disclaimer
- Version note
- Next step CTA

## 5. Required Wording Discipline

Writers must:
- use "public preview" where content is early
- use "draft" where content is not reviewed
- use "future" where feature is not live
- use "requires review" where specialist review is needed
- use "learning support" for educational material
- use "may" where meaning is context-dependent
- use "approximate meaning" for vocabulary where needed
- use "not live yet" for unavailable features
- use "not legal advice" for legal material
- use "not therapy/diagnosis" for mental wellness material
- use "not tafsir/fatwa/fiqh ruling" for Quran/Seerah material
- use "no guarantee" for earning, fluency, certification, outcome, or transformation claims

Writers must not:
- present vision as current reality
- present draft as final
- present prototype as production
- present learning as professional service
- present reflection as therapy
- present legal literacy as legal advice
- present spiritual reflection as religious authority
- present earning education as financial advice
- present vocabulary as verified if not reviewed
- present AI/DLAS as live runtime if not implemented and governed

## 8. Page-Specific Writing Rules

### A. Language & Knowledge Hub Home

Rules:
- introduce the hub as learning/reflection gateway
- do not claim full curriculum is live
- mention Bangla/English/Arabic as paths/candidates
- clearly label public preview

### B. Start With One Word

Rules:
- present one-word beginning as reflection doorway
- do not present as diagnosis/intake/scoring
- no hidden AI implication
- include "one word is a beginning, not a full story"

### C. Trilingual Vocabulary Preview

Rules:
- use approximate meaning language
- mark vocabulary as unverified/public-preview unless reviewed
- do not include religious authority wording
- no fluency guarantee

### D. Knowledge / Truth / Human / Responsibility / Learning / Action Pages

Rules:
- concept explanations should be simple
- include Bangla and Arabic terms only with review label
- avoid philosophical overclaim
- use reflection/action balance

### E. How to Learn Preview

Rules:
- educational only
- no scientific claim unless sourced
- no guaranteed mastery
- active recall/spaced repetition as learning methods, not miracles

### F. How to Earn Preview

Rules:
- no income guarantee
- no financial advice
- no job/client promise
- present as learning and experimentation

### G. Emotional Intelligence Preview

Rules:
- no therapy/diagnosis/treatment
- no mental health authority
- reflection and education only
- crisis/emergency boundary if distress appears

### H. Legal Hub / Laws of Bangladesh Pages

Rules:
- adjacent future/internal unless verified
- not direct Language Hub core page
- no legal advice
- official source verification required
- lawyer review required before public use
- hotline/contact verification required

### I. Quran/Seerah Pages

Rules:
- future/internal unless religious-boundary and source review complete
- no tafsir/fatwa/fiqh
- no tajweed instruction
- no unsourced religious or historical claims

### J. AI/DLAS Pages

Rules:
- future concept unless evidenced live
- no runtime claim
- no validated assessment claim
- no automated decision/scoring/diagnosis claim
- state consent/privacy/validation boundaries

## 9. CTA Writing Rules

Allowed CTAs:
- "Continue to Deeper Study"
- "Read the next public-preview page"
- "Reflect on one small action"
- "Save this as a learning note"
- "Return to Source of Truth"
- "Check the status boundary"
- "Register interest when available" only if manual interest path exists

Not allowed CTAs:
- "Join paid lab now"
- "Get certified now"
- "Start AI analysis"
- "Begin DLAS assessment"
- "Pay now"
- "Get legal help now"
- "Receive therapy"
- "Guarantee your income"
- "Start recovery treatment"

## 10. Status and Disclaimer Placement Rules

Every page must place:
- status label near the title
- level label near the title
- review status near the title or intro
- disclaimer before high-risk content
- "what this page is / is not" before examples
- version note near the end
- CTA after disclaimer and page content, not before boundary

## 11. Bilingual / Multilingual Writing Rules

- English/Bangla-ready layout should be supported.
- Bangla should be plain and citizen-friendly.
- English should be simple and non-technical where public.
- Arabic should not be used as authority unless reviewed.
- Arabic transliteration must be marked draft unless reviewed.
- Terms should not be forced into one-to-one equivalence.
- Use "approximate meaning" where needed.
- Keep original script visible where possible.
- Do not use Arabic religious terms as rulings.

## 13. Public Page Final Review Questions

Before any draft is approved, answer:

- What is the strongest claim on this page?
- What evidence supports it?
- Is the page's status label accurate?
- Does the page imply a feature is live?
- Does the page imply professional authority?
- Does it need specialist review?
- Does it include required disclaimer?
- Could a vulnerable person misunderstand it?
- Does it match current public status?
- Does it preserve Claim <= Evidence?
