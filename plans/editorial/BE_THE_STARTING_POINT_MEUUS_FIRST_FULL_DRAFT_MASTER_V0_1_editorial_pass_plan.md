# Editorial Pass Plan

## Governance Boundary

This plan does not approve publication.

This plan does not make the manuscript canonical.

This plan does not amend Frozen governance.

This plan does not rewrite the manuscript yet.

This plan only defines the order and rules for future editorial work.

Any public use still requires evidence review, sensitivity review, and public-expression review under Documents 05, 06, and 07.

## Editorial Objectives

- Preserve founder voice without exposing unsafe private material.
- Preserve truth discipline.
- Preserve Claim <= Evidence.
- Separate vision from claim.
- Separate prototype from production.
- Separate book voice from governance authority.
- Separate personal origin from validated evidence.
- Separate reflection from therapy/diagnosis.
- Separate legal literacy from legal advice.
- Separate spiritual reflection from religious authority.
- Make public-safe excerpts possible later.
- Create a cleaner manuscript version path.

## Editorial Pass Sequence

### Pass 0 - Preservation Pass

Purpose:

Preserve original manuscript as source material.

Actions:

- do not overwrite original
- record file name and version indicators
- keep original unchanged
- maintain source record

Output:

Original preserved.

### Pass 1 - Structure Pass

Purpose:

Check chapters, parts, headings, duplication, ordering, and manuscript flow.

Actions:

- verify table of contents
- detect repeated sections
- identify missing transitions
- identify chapter-level purpose

Output:

Structure notes.

### Pass 2 - Claim Boundary Pass

Purpose:

Identify and soften claims using the claim-risk register.

Actions:

- mark live/future/platform/AI/DLAS/research/legal/therapy/religious claims
- assign status labels
- rewrite guarantees into bounded language

Output:

Claim-safe draft notes.

### Pass 3 - Privacy and Sensitivity Pass

Purpose:

Protect founder, family, private people, recovery, legal, religious, and emotional material.

Actions:

- mark private/archive material
- anonymize or abstract where needed
- remove unnecessary identifiable details
- protect dignity

Output:

Sensitivity-safe notes.

### Pass 4 - Evidence and Provenance Pass

Purpose:

Check whether factual/public/operational claims have evidence.

Actions:

- map claims to evidence
- mark unknowns
- mark future/planned
- mark prototype/manual/demo/live status

Output:

Evidence map.

### Pass 5 - Public Expression Pass

Purpose:

Prepare possible public-facing excerpts without creating authority.

Actions:

- select safe themes
- add labels/disclaimers
- map to surfaces
- ensure public language does not overclaim

Output:

Public excerpt draft candidates later.

### Pass 6 - Book Voice Pass

Purpose:

Improve readability while preserving authenticity.

Actions:

- reduce repetition
- clarify doctrine
- balance emotion and discipline
- improve transitions
- avoid inflated language

Output:

Cleaner manuscript draft.

### Pass 7 - Bilingual/Localization Pass

Purpose:

Prepare later English/Bangla alignment if needed.

Actions:

- mark terms requiring consistent translation
- preserve Islamic phrases with care
- avoid meaning loss

Output:

Localization notes.

### Pass 8 - Version Reconciliation Pass

Purpose:

Resolve version mismatch later.

Actions:

- compare file name v0.1 with internal Opening Draft v0.2
- choose version path later
- do not resolve now

Output:

Version reconciliation note.

### Pass 9 - Approval Routing Pass

Purpose:

Define approval route before public use.

Actions:

- founder review
- architecture review
- evidence review
- sensitivity review
- public-expression review
- qualified review where needed

Output:

Approval checklist.
