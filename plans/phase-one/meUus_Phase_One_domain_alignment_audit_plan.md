## 2. Purpose of This Audit Plan

This audit plan defines how Phase One will review and align the public meaning, claims, boundaries, and route responsibilities of the meUus domains before any further public announcement, indexing, DNS/redirect changes, or production-ready claims.

## 5. Audit Questions

For each domain, the later audit must ask:

- What does this domain currently claim?
- What should this domain claim?
- What must it not claim?
- What is live?
- What is preview?
- What is draft?
- What is future?
- What user action is allowed?
- What user action is not allowed?
- What privacy or sensitive data risk exists?
- What overclaim risk exists?
- What correction is needed?

## 6. Cross-Domain Alignment Checks

- Preview/public/production separation.
- AI/DLAS/account/payment/certification boundary consistency.
- Privacy/contact consistency.
- Noindex/indexing consistency.

## 12. Risks and Safeguards

| Risk | Safeguard |
| --- | --- |
| meUus appears more complete than it is | Require status labels and Claim <= Evidence wording across domains. |
| User thinks AI/DLAS/app/accounts are live | Audit all AI/DLAS/app/account/payment/certification claims and mark future/not approved where needed. |
| rujbel.org appears as official institution | Keep rujbel.org as Founder / Human / Origin / Living Archive, not meUus institutional authority. |
| Founder private data leaks | Do not publish private founder/family/contact data; require public-safety review. |
| Language Hub appears verified/authoritative | Preserve public-preview, educational, review-bounded, and no authority language. |
| Domains contradict each other | Compare role, status, navigation, and claims across all domains before corrections. |
| Public announcement happens before correction | Maintain announcement hold until audit findings and corrections are reviewed. |
| Indexing happens before review | Preserve noindex/indexing caution unless a separate indexing approval is recorded. |
| `reports/recommendations/` is accidentally committed | Leave `reports/recommendations/` untouched unless separately instructed. |
| Document 08 is created prematurely | Do not create Document 08; use scoped Phase One records and plans. |
