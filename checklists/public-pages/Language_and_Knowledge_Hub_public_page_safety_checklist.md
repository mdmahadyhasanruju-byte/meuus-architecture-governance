## 2. Governance Boundary

- This checklist does not approve publication.
- This checklist does not approve implementation.
- This checklist does not approve deployment.
- This checklist does not create final copy.
- This checklist does not make pages canonical.
- This checklist does not verify vocabulary.
- This checklist does not verify Arabic/Quran/Seerah content.
- This checklist does not verify legal, financial, psychological, medical, recovery, religious, or scientific claims.
- This checklist only defines safety gates before public use.

Core rule:
No public page should be published unless its claim strength, status label, source/provenance status, risk boundary, and required disclaimer are clear.

## 3. Universal Pre-Publication Checklist

### Identity and Status

- [ ] Page has title
- [ ] Page has status label
- [ ] Page has level label
- [ ] Page has review status
- [ ] Page has source/provenance status
- [ ] Page has version note
- [ ] Page has last reviewed date or "not reviewed yet"

### Claim Safety

- [ ] Page does not claim more than evidence supports
- [ ] Page separates live from draft/prototype/future
- [ ] Page avoids guaranteed outcomes
- [ ] Page avoids professional authority unless reviewed
- [ ] Page avoids "complete," "validated," "live," "official," or "certified" unless evidenced
- [ ] Page does not imply accounts, payments, AI runtime, DLAS runtime, cloud storage, subscriptions, rewards, dashboards, or automated decisions are live

### Source Safety

- [ ] Page identifies source material category
- [ ] Page distinguishes founder draft from reviewed material
- [ ] Page distinguishes public content from internal/private content
- [ ] Page has source/provenance record where needed
- [ ] Page avoids unsourced legal/religious/scientific claims

### Human Safety

- [ ] Page does not provide diagnosis
- [ ] Page does not provide therapy
- [ ] Page does not provide emergency care
- [ ] Page does not provide legal advice
- [ ] Page does not provide financial advice
- [ ] Page does not provide religious ruling
- [ ] Page provides referral/disclaimer where high-risk content appears

## 6. Go / Hold / Block Decision Model

### GO — Public Preview Candidate

Allowed only if:
- low risk
- labels complete
- disclaimers complete
- no specialist claim
- no unverified authority
- no runtime overclaim
- source/provenance acceptable for public preview

### HOLD — Needs Review

Use when:
- vocabulary unverified
- source unclear
- wording could imply authority
- page needs specialist review
- law/religion/mental health/finance/recovery content appears

### BLOCK — Not Approved for Public Use

Use when:
- claims live feature without evidence
- implies legal/religious/medical/financial authority
- uses unverified hotline/contact info
- includes private founder/internal material without approval
- promises income, certification, therapy, legal outcome, recovery, or AI/DLAS judgment
- contradicts current public status boundaries
