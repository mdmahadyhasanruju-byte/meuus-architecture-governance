## 2. Governance Boundary

- This review does not approve implementation.

## 5. Source Type Classification

| Source Type | Allowed Use | Risk Level | Citation Required? | Reviewer Needed? | Notes |
|---|---|---|---|---|---|
| Dictionary | Meaning reference and spelling support | Medium | Yes | Yes | Must avoid one-to-one equivalence overclaim |
| Academic textbook | Structured language or concept reference | Medium | Yes | Yes | Edition and page/chapter should be recorded where possible |
| Language learning textbook | Beginner/intermediate learning support | Medium | Yes | Yes | Useful for pedagogy, but not sole authority |
| Arabic grammar reference | Arabic grammar, root, case, number, gender support | High | Yes | Yes | Requires Arabic reviewer |
| Quran translation | Translation reference only | Critical | Yes | Yes | Translation source must be named; not tafsir by itself |
| Tafsir source | Quran interpretation source | Critical | Yes | Yes | Requires religious-boundary and qualified review |
| Hadith/Seerah source | Historical/religious source material | Critical | Yes | Yes | Must record source type and reliability status where applicable |
| Scholarly article | Research or academic support | High | Yes | Yes | Claims must be bounded and contextualized |
| Teacher-created content | Lesson, explanation, example sentence | Medium | Yes | Yes | Teacher role and review date should be recorded |
| Founder-created draft | Origin draft or concept framing | High | Internal citation required | Yes | Not verified; must be labeled Founder Draft |
| AI-generated draft | Draft candidate only | High | Internal note required | Yes | Never treated as verified without human review |
| Website/blog | Supplemental public source | High | Yes | Yes | Must assess quality and stability |
| Community contribution | Suggested word/correction/example | High | Yes | Yes | Must be reviewed before acceptance |
| User-submitted correction | Correction candidate | High | Internal record required | Yes | Requires reviewer approval and version history |

## 6. Provenance Risk Table

| Risk Area | Example | Severity | Required Control | Current Decision |
|---|---|---|---|---|
| Vocabulary copied without source | Vocabulary list imported without citation | High | Source type, source title, and citation required | Not approved |
| AI-generated vocabulary treated as verified | AI draft words published as accurate | High | Human linguistic review and AI draft label | Not approved |
| One-to-one translation without context | Arabic word mapped to one English word only | High | Approximate meaning and context notes | Needs review |
| Arabic religious term without source | Spiritual term explained from memory | Critical | Source/provenance and religious-boundary review | Not approved |
| Quran translation without translation source | Verse translation shown without translator/source | Critical | Translation source required | Not approved |
| Seerah claim without reference | Historical event summarized without source | Critical | Seerah source/provenance review | Not approved |
| Example sentence containing unsourced religious meaning | Sentence teaches doctrine through vocabulary example | Critical | Religious-boundary review and source tracking | Not approved |
| Bangla transliteration inconsistent | Same Arabic sound rendered multiple ways | Medium | Transliteration standard and correction log | Needs review |
| Arabic Bangla-script pronunciation invented | Bangla-script Arabic pronunciation made without reviewer | High | Arabic/Bangla reviewer approval | Not approved |
| Duplicate terms with conflicting meanings | Same term appears twice with different meanings | Medium | Deduplication and version reconciliation | Needs review |
| Professional/legal/business terms used without domain review | Legal/business vocabulary implies advice | High | Domain reviewer and disclaimer | Not approved |
| User-submitted correction accepted without reviewer | Public correction applied automatically | High | Reviewer approval and correction log | Not approved |
| Old source/version conflict | Earlier meaning conflicts with later corrected source | Medium | Version history and correction policy | Needs review |
| Public page missing provenance status | Vocabulary table appears authoritative | High | Visible source/provenance status | Not approved |

## 7. Public Citation/Provenance Rules

Before public use:

- Every vocabulary table must include source/provenance status.
- Quran translations must identify translation source.
- Seerah references must identify source/provenance.
- Arabic religious terms must be labeled "language learning support."
- AI-generated draft content must be reviewed before publication.
- Founder-created definitions must be labeled as draft/reflection unless externally sourced.
- User corrections must be reviewed before acceptance.
- Public pages should not expose internal reviewer names unless consent exists.
- Public-facing citations can be concise, but internal source records must be complete.

## 8. Correction and Versioning Policy

Every vocabulary item must support:

- correction date
- correction reason
- old value
- new value
- reviewer
- version number
- public change note if already published
- rollback note if needed

Version labels:
- v0.1 Founder Draft
- v0.2 Source Located
- v0.3 Linguistic Review
- v0.4 Corrected Draft
- v0.5 Public Preview Candidate
- v1.0 Public Preview Approved
- Canonical Candidate only after governance approval

## 9. Public Page Vocabulary Rules

Before any vocabulary page becomes public, it must include:

- page status label
- vocabulary review status
- source/provenance status
- "Learning Support" label
- "Not Fluency Guarantee" label
- "Not Tafsir/Fatwa" label where Quran/Seerah terms appear
- "Requires Review" label where applicable
- approximate meaning disclaimer where needed
- no claim of complete dictionary authority
- no claim of religious interpretation
- no claim of certification or professional fluency
